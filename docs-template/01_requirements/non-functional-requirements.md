# 非機能要件（Non-Functional Requirements）

## 1. 概要
- 本書は、本システム全体に関わる非機能要件を定義する。

## 2. 性能 / パフォーマンス
- 応答時間、処理時間、スループットなどに関する要件を記載する。

## 3. 可用性 / 信頼性
- システムの稼働時間や障害時の振る舞いなど、可用性・信頼性に関する要件を記載する。

## 4. セキュリティ
- 認証・認可、データ保護、不正利用防止などのセキュリティ要件を記載する。

## 5. 監査 / ログ
- 監査ログや操作ログの取得方針、保存期間などを記載する。

## 6. 運用 / 監視
- 監視対象、アラート、運用体制などの運用・監視方針を記載する。

## 7. スケーラビリティ
- 将来的な利用者増加やデータ量増加に対する拡張性の要件を記載する。

## 8. 保守性 / 移行
- 保守・変更のしやすさや、システム移行・データ移行に関する要件を記載する。

## 9. ユーザビリティ / アクセシビリティ
- 操作性やアクセシビリティに関する要件を記載する。
