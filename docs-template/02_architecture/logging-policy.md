# ログ出力方針

## 位置づけ
本ドキュメントは、システムの非機能要件に基づき、
ログ出力に関する基本方針および実現方式の考え方を示すものである。

## 非機能要件との関係
ログ出力は、以下の非機能要件を満たすことを目的とする。

- 障害発生時の原因特定
- 業務操作のトレーサビリティ確保
- 不正操作や異常挙動の検知
- 運用時の状況把握および監視

## 基本方針
- ログ出力は横断的関心事として扱う。
- 業務ロジックからログ処理を分離する。
- Controller、サービス、リポジトリ等の業務コード内に、
  デバッグログを除いてログ出力のための明示的な処理を記述してはならない。
- デバッグログは開発・検証用途に限定して使用すること。
- ログは集中管理され、自動的に出力される仕組みを用いる。

## 実現方式の考え方
- ログ出力は、横断的に処理を適用できる仕組み
  （例：AOP、Interceptor 等）を用いて実現する。
- 具体的な技術やフレームワークは、本テンプレートでは固定しない。

## 個別システムでの具体化
- 本テンプレートに基づき、各システム固有のログ出力方針および方式は、
  アーキテクチャ設計（docs/02_architecture）にて定義すること。
