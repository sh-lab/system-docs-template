# データアーキテクチャ設計方針

## 1. 目的
本書は、本システムにおけるデータ構造およびデータアーキテクチャ設定義する。

## 2. 適用範囲
本方針は、本システムが扱う永続データに適用する。

## 3. データ設計の基本方針
- 原則として正規化を行い、データの整合性を担保する。
- 冗長性は必要性が明確な場合にのみ限定的に許容する。
- 主キー・外部キーを明確に定義し、参照関係を構造として表現する。

## 4. 同時更新制御の方針
- 同時更新が発生し得る業務データについては、楽観的排他制御を採用する。
- 排他制御違反時は業務例外として扱い、再取得・再試行を基本とする。

## 5. 監査データの方針
- 業務データの作成・更新に関する監査情報を保持する。
- 監査情報はアプリケーション側で自動設定し、利用者入力は不可とする。

## 6. データ変更管理の考え方
- データ構造の変更は、履歴が追跡可能な方法で管理する。
- 変更適用手順は再現可能でなければならない。

## 7. 補足
- テーブル物理設計やカラム定義は設計ドキュメント側に記載する。
- 本方針を変更する場合は ADR に記録する。
