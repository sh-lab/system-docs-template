# クエリサービス設計書（Query Service）

## 1. サービスID・名称
- ID：［QS-XXX］
- 名称：［○○QueryService］

---

## 2. 役割と責務
- 本サービスは **読み取り専用（副作用なし）** の検索・取得・一覧・集計を担当する。
- 画面／API に返す **DTO（表示用データ）** を組み立てる。
- トランザクション境界は管理しない（Application Service に委譲する）。
- 書き込み（状態変更）や業務ルールの判断は行わない。

---

## 3. 目的・スコープ
- **目的**：［このクエリでユーザーが得る情報価値を1行で記載］
- **対象データ**：［例：Asset の一覧／詳細／サマリ］
- **利用シーン**：［例：SCR-Asset-List、外部API /assets など］

---

## 4. 入力仕様（検索条件）

### 4.1 条件一覧
| 条件名 | 型 | 必須 | マッチ種別 | 備考 |
|--------|----|------|------------|------|
| name | string | 任意 | 前方一致／部分一致／完全一致 | 最大100文字 |
| status | enum | 任意 | 完全一致 | DRAFT/ACTIVE/RETIRED |
| updatedFrom | datetime | 任意 | 範囲下限 | ISO 8601 |
| updatedTo | datetime | 任意 | 範囲上限 | ISO 8601 |

### 4.2 バリデーション
- 形式・必須の検証は **Controller 層で完了**している前提。
- 条件間整合（from ≤ to 等）だけをここで補足しても良い。

---

## 5. 並び替え・ページング

### 5.1 ソート
- 既定：`updatedAt DESC`
- 許可：`name ASC/DESC`, `assetId ASC/DESC`

### 5.2 ページング
- 既定ページサイズ：50（最大 200）
- 返却メタ：`page`, `size`, `total`, `hasNext`, `hasPrev`

---

## 6. 出力仕様（DTO）

### 6.1 一覧 DTO（サマリ）
| 項目名 | 型 | 必須 | 説明 |
|--------|----|------|------|
| assetId | string | ○ | エンティティ識別子 |
| name | string | ○ | 名称 |
| status | string | ○ | 表示用ラベル |
| updatedAt | datetime | ○ | 最終更新日時 |

### 6.2 詳細 DTO（必要時のみ）
| 項目名 | 型 | 必須 | 説明 |
|--------|----|------|------|
| assetId | string | ○ |  |
| name | string | ○ |  |
| status | string | ○ |  |
| attributes | object | 任意 | 追加属性 |

> ルール：DTO は **表示／API応答に必要な最小項目**に限定する。  
> 集約・結合が複雑な場合は **Repository（Query）側で組み立て**、業務判断は持ち込まない。

---

## 7. 処理フロー概要（擬似コード可）
> 実装詳細や制御構文は記載しない。流れのみを示す。

1. 入力条件を受け取る  
2. ソート・ページング既定値を適用する  
3. Query Repository を呼び出して結果を取得する  
4. 取得結果を DTO にマッピングする  
5. ページングメタ情報を付与して返却する  

---

## 8. 使用するコンポーネント
- **Query Repository**：［対象ビュー／テーブル群の検索・集計を担当］
- **（任意）変換ユーティリティ／マッパー**：DTO 変換に限定（副作用なし）

> ルール：**Entity Repository を更新目的で呼ばない**。  
> 必要に応じて参照のみ可（ただし副作用禁止）。

---

## 9. 例外とエラー方針
- 検索条件の論理不整合：**業務例外**（400）  
- データアクセス障害：**システム例外**（500／汎用）  
- 例外の分類・UI/API への変換は **Application Service の責務**。

参照：`02_architecture/error-handling.md`

---

## 10. 非機能要件への配慮（該当時）
- 性能：P95 < 1 秒（一覧／page=50）  
- 安定性：ソートキーは **二次キー**まで指定し、順序の安定性を担保  
- 転送量：必要項目に限定し、列挙値は表示用ラベルを返す

---

## 11. 禁止事項
- 副作用（登録・更新・削除）を行わない  
- Command Service を呼び出さない  
- 業務ルールの判断を持ち込まない（表示整形のみ可）  
- トランザクションを開始しない（境界は Application Service）

---

## 12. 補足
- 集計／結合が重いレポート用途は **別クエリとして個票化**を検討する。  
- 例外的方針は ADR に記録する。
